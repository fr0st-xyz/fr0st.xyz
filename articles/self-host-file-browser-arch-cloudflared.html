How to Self-Host File Browser on Arch Linux with Cloudflared Tunnels

Views:



File Browser is a lightweight web-based file manager that lets you access and manage your files from any device. In this guide, I'll show you how to host it on Arch Linux using Cloudflared Tunnels.

A quick note before we dive in: vulnerabilities always exist, and nothing on the internet is completely secure. I personally recommend not storing personal or sensitive information in your File Browser. Of course, it's your choice, and I'm not responsible if anything happens to your File Browser after following this guide.

1. Prerequisites

Before we start, make sure you have the following:

Arch Linux installed on a VPS, another PC, or any computer you want to use

A domain you control

Cloudflare account (free plan works)

A little bit of experience with Linux commands

2. Point your domain to Cloudflare

If you've never pointed a domain to Cloudflare before, check out this step-by-step guide for a detailed walkthrough.

3. Setup File Browser

# Create directories

First, lets create some directories to keep things organized. This is how I like to set up File Browser before running it. Open your terminal and run these commands:

Code 
mkdir -p /home/$USER/.local/share/filebrowser/branding/img/icons
mkdir -p /home/$USER/Desktop/filebrowser-data

# Install File Browser

After creating the directories, lets now install File Browser. Run the following command in your terminal:

Code 
curl -fsSL https://raw.githubusercontent.com/filebrowser/get/master/get.sh | bash

# Initialize File Browser database

Next, lets initialize the File Browser database in the directory we just created. Run this command to create a new configuration file and set up the database:

Code 
filebrowser -d /home/$USER/.local/share/filebrowser/filebrowser.db config init

# Create admin user

Now, let's create a new admin user for File Browser. Run the following command, replacing admin with your desired username and "YourChosenP@ss" with a secure password of your choice:

Code 
filebrowser -d /home/$USER/.local/share/filebrowser/filebrowser.db users add admin 'YourChosenP@ss' --perm.admin

To verify that the user was created successfully, run:

Code 
filebrowser -d /home/$USER/.local/share/filebrowser/filebrowser.db users ls

# Custom branding

Give your File Browser a custom name (replace fr0st with any name you like) and use the branding folder we created earlier. Run:

Code 
filebrowser -d /home/$USER/.local/share/filebrowser/filebrowser.db \
  config set --branding.name "fr0st" \
             --branding.files "/home/$USER/.local/share/filebrowser/branding"

This sets the interface title to your chosen name and uses the branding folder we created for custom icons/images. If you want to update your favicon or other branding assets, you can check the folder structure here and for more detailed information, read the File Browser documentation on custom branding

# Use reCAPTCHA for Login

It is recommended to add a reCAPTCHA to your File Browser login page to prevent automated login attempts. By default, File Browser uses Google reCAPTCHA, and this guide will follow that setup. If you prefer to use a different provider, you can find instructions in the official FileBrowser documentation here:

To set up reCAPTCHA for your File Browser login, follow these steps:

1. Go to the Google reCAPTCHA Admin Console
2. Create a new site and add these:

Label: FileBrowser

reCAPTCHA type: reCAPTCHA v2 ‚Üí "I'm not a robot" Checkbox

Domain: Enter the subdomain data because that's what we'll use in this guide, then replace yourdomain.com with your actual domain e.g., data.yourdomain.com.

Here's a screenshot showing how it should look in Google reCAPTCHA:

Click Submit.

3. Copy your keys:

Site Key ‚Üí used in the browser for the captcha widget

Secret Key ‚Üí used by File Browser to verify captcha responses

4. Run the following command to set your File Browser keys (make sure to replace YOUR-SITE-KEY and YOUR-SECRET-KEY with the keys you copied from Google):
Code 
filebrowser -d /home/$USER/.local/share/filebrowser/filebrowser.db \
  config set --auth.method=json \
             --recaptcha.key YOUR-SITE-KEY \
             --recaptcha.secret YOUR-SECRET-KEY

# Run File Browser locally

Finally, lets run File Browser on your local machine. Use the following command to start it on localhost at port 8081, using the files from the directory we created:

Code 
filebrowser -d /home/$USER/.local/share/filebrowser/filebrowser.db -a 127.0.0.1 -p 8081 -r /home/$USER/Desktop/filebrowser-data

Once its running, open your browser and go to: http://127.0.0.1:8081/

Note

If you see the reCAPTCHA, saying "Localhost is not in the list of supported domains for this site key." That's normal, you need to run File Browser online on your subdomain for the keys to work.

4. Setup Cloudflared Tunnels

# Install Cloudflared

Lets install Cloudflared so you can run it online on your subdomain and create a tunnel. Open a new terminal and run:

Code 
sudo pacman -S cloudflared

# Create a Cloudflared Tunnel

First, log in to Cloudflare on your system by running:

Code 
cloudflared tunnel login

This will open a browser window asking you to log in to your Cloudflare account. Once logged in, select your domain and click Authorize. Your system is now authorized to create and manage tunnels.

After logging in, run this command to create a tunnel for File Browser:

Code 
cloudflared tunnel create filebrowser

When you run the tunnel command, it prints a UUID in the terminal (something like 12345678-90ab-cdef-1234-567890abcdef). Copy that you'll need it for your DNS. ..but if you're like me and sometimes clear the terminal by mistake (I feel this urge to always clear it after seeing so much shit on the screen), just run:

Code 
cloudflared tunnel list

This will show all your tunnels so you can copy the UUID for the filebrowser tunnel.

# Point your subdomain to the Cloudflared tunnel

Now that you have the UUID, Open your Cloudflare Dashboard, go to DNS ‚Üí Add record, and enter the following details:

Type: CNAME

Name: data ‚Üí data.yourdomain.com

Target: YOURUUID.cfargotunnel.com (replace YOURUUID with the UUID you copied from your tunnel)

Proxy status: Proxied (orange cloud)

Click Save

Here's how it should look (My UUID is hidden, but you get the idea):

# Configure the tunnel

I'll use vim to create the configuration file for the Cloudflared Tunnel, but you can use any text editor you prefer:

Code 
vim /home/$USER/.cloudflared/filebrowser-tunnel.yml

Then add the following content, replacing YOUR-UUID-HERE with your UUID, data.yourdomain.com with your actual domain, and $USER with your local username:

Code 
tunnel: YOUR-UUID-HERE
credentials-file: /home/$USER/.cloudflared/YOUR-UUID-HERE.json

ingress:
  - hostname: data.yourdomain.com
    service: http://127.0.0.1:8081
  - service: http_status:404

Don't forget to save the file brodie. üòâ

# Run the tunnel

Now, run the tunnel in another terminal using the command below to make your File Browser accessible online:

Code 
/usr/bin/cloudflared tunnel --config /home/$USER/.cloudflared/filebrowser-tunnel.yml run filebrowser

And finally visit https://data.yourdomain.com and you should see File Browser login.

5. Systemd service (File Browser + Tunnel)

# Create Systemd Service

Lets create a systemd service to run File Browser together with the Cloudflared Tunnel always on startup, so you don't have to open terminals and start them manually every time. I'll use vim, but you can use your preferred text editor:

Code 
sudo vim /etc/systemd/system/filebrowser.service

Then paste the following content, but replace $USER with your actual Linux username (since $USER doesn't work in systemd service files):

Code 
[Unit]
Description=FileBrowser + Cloudflare Tunnel
After=network.target

[Service]
Type=simple
User=$USER
WorkingDirectory=/home/$USER
ExecStart=/bin/bash -c "/usr/bin/cloudflared tunnel --config /home/$USER/.cloudflared/filebrowser-tunnel.yml run filebrowser & \
filebrowser -d /home/$USER/.local/share/filebrowser/filebrowser.db -a 127.0.0.1 -p 8081 -r /home/$USER/Desktop/filebrowser-data"
Restart=on-failure
RestartSec=5

[Install]
WantedBy=multi-user.target

# Enable and Start the service

Run the following commands to enable and start the service:

Code 
sudo systemctl daemon-reload
sudo systemctl enable filebrowser.service
sudo systemctl start filebrowser.service

To check if everything is working, run:

Code 
sudo systemctl status filebrowser.service

If it says Active: active (running), you're all set ü•π (yes its finally over) that's it!

6. Updating File Browser

# How to update FileBrowser

To get the latest version of File Browser, just install it again:

Code 
curl -fsSL https://raw.githubusercontent.com/filebrowser/get/master/get.sh | bash

Then restart the service to apply the update:

Code 
sudo systemctl restart filebrowser.service
